D / Errors / DOS-related Problems ]" border=0> In general, any dos.library function which fills in a structure for you (for example, Examine() ), requires that the structure be longword aligned. In most cases, the only way to insure longword alignment in C is to dynamically allocate the structure. Unless documented otherwise, dos.library functions may only be called from a process, not from a task. Also note that a process's pr_MsgPort is intended for the exclusive use of dos.library. (The port may be used to receive a WBStartup message as long as the message is GetMsg() 'd from the port before DOS is used. [Back to Amiga Developer Docs ]
