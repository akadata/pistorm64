TITLE: 24 / Messages / Putting a Message

24 / Messages / Putting a Message ]" border=0> A message is delivered to a given destination port with the PutMsg() function. The message is queued to the port, and that port's arrival action is invoked. If the action specifies a task signal or a software interrupt, the originating task may temporarily lose the processor while the destination processes the message. If a reply to the message is required, the mn_ReplyPort field must be set up prior to the call to PutMsg(). Here is a code fragment for putting a message to a public port.
